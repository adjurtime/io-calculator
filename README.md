# IO Calculator

浏览器端投入产出（Input–Output, IO）分析工具，用于导入单区域 IO（SRIO）或按区域连续排列的多区域 IO（MRIO）矩阵，完成数据校验、系数与逆矩阵计算、卫星账户扩展、产业关联分析及结果导出。

当前版本：`2.0.0-alpha.2`

> English summary: A client-side IO analysis prototype for matrix validation, Leontief/Ghosh calculations, satellite-account multipliers, sector aggregation, and local result export.

## 当前状态

当前版本是 v2 的第二阶段 alpha，重点是可信计算、可取消后台执行和明确的浏览器边界；仍不应直接作为正式论文结果的唯一计算依据。

- 内置三部门示例可在开发服务器中完成“输入—校验—计算—结果”流程。
- `npm run build` 已通过 TypeScript 与 Vite 8 生产构建。
- 7 个测试文件包含 23 项离线测试，覆盖 SRIO 黄金算例、多卫星/多最终需求足迹、严格导入、MRIO 合并、规模限制、HTML 转义和矩阵求逆。
- 缺失 `x`、空单元格、非数值、ragged 矩阵、零产出和 MRIO 区域错配会阻止计算。
- 足迹结果使用“卫星账户 × 最终需求类别”契约，计算失败不会再进入结果页。
- 密集矩阵计算在 Web Worker 中运行，可由用户取消；浏览器 alpha 明确限制为最多 500 个部门。
- Leontief 和 Ghosh 结果包含逆矩阵残差及无穷范数条件估计。

正式研究中仍应使用独立程序复核关键矩阵、数据口径和结果。下一阶段将重点处理线性方程求解、Excel 依赖替换和更完整的可访问性审计。

## 已实现功能

- 四步工作流：数据输入 → 数据校验 → 指标计算 → 结果导出。
- 内置三部门示例数据。
- Excel、CSV、TXT、DAT 和有限 MAT 文件导入。
- 从 Excel 或文本按数据类型逐项粘贴。
- Excel 多 Sheet 手动映射到 `Z`、`x`、`Y`、`VA`、`F`。
- 维度、零产出、负值和会计恒等式校验。
- SRIO 与手动 MRIO 区域配置。
- 连续等量合并或按部门编号自定义合并。
- 直接消耗系数、分配系数、Leontief 逆和 Ghosh 逆。
- 增加值系数、投入率、增加值率及乘数。
- 卫星强度、足迹乘数和最终需求足迹。
- 前向/后向产业关联及关键产业识别。
- 矩阵表格和热力图。
- Excel 与 JSON 下载。
- WIOD、EXIOBASE、GTAP、OECD ICIO、EORA、CEADs 等数据库的外部导航链接。

数据库链接只用于导航；应用不会自动下载或转换这些数据库的数据。

## 快速开始

### 环境要求

- Node.js 18 或更高版本；
- npm；
- 支持 ES2020、File API 和 Blob 下载的现代浏览器。

### 本地开发

```bash
git clone https://github.com/adjurtime/io-calculator.git
cd io-calculator
npm ci
npm run dev
```

开发服务器默认尝试使用 `http://localhost:5173`，并自动打开浏览器。

### 构建与预览

```bash
npm run build
npm run preview
```

`npm run build` 会先执行 TypeScript 检查，再由 Vite 构建到 `dist/`。

### 测试

```bash
npm test -- --run
```

当前测试套件包含 23 项离线测试。新增或修改计算逻辑时，应同时添加对应的黄金算例或失败用例。

## 推荐使用流程

1. 点击“加载示例”熟悉界面，或上传自己的数据。
2. 至少提供 `Z` 和真实的 `x`；按需补充 `Y`、`VA` 和 `F`。
3. 如使用 MRIO，设置区域数、每区域部门数和区域名称。
4. 进入数据校验，检查维度、会计平衡和所有警告。
5. 如需降低维度，在校验页执行部门合并。
6. 选择指标和容差，运行计算。
7. 在表格或热力图中检查结果。
8. 下载 Excel 或 JSON，并用独立程序复核关键结果。

页面刷新或点击“重置”会清除当前状态。应用目前没有项目保存、自动恢复或撤销功能。

## 数据模型与基本假设

令总部门数为 $n$。

| 数据 | 是否必需 | 维度 | 含义 |
|---|---:|---:|---|
| `Z` | 是 | $n\times n$ | 中间投入矩阵，$z_{ij}$ 表示部门 $i$ 向部门 $j$ 提供的中间产品 |
| `x` | 是 | $n$ | 各部门总产出 |
| `Y` | 否 | $n\times k$ | 最终需求矩阵，$k$ 为最终需求类别数 |
| `VA` | 否 | $m\times n$ | 增加值分项 |
| `F` | 否 | $p\times n$ | 直接卫星账户，如碳排放、能源或用水 |
| 部门名称 | 否 | $n$ | 未提供时使用“部门1、部门2……” |
| 区域名称 | 否 | 区域数 | 用于 MRIO 区域汇总 |

数据使用者需要保证：

- `Z`、`x`、`Y` 和 `VA` 使用一致的货币单位、价格口径、年份和进口处理方法；
- `F` 与 `Z`、`x` 的部门顺序完全一致；
- 单元格只包含可解析的数值，不包含单位字符或千位分隔文本；
- 矩阵不存在缺列、错位或不等长的行；
- 币种、价格、通胀、单位和部门分类转换已在导入前完成。

### MRIO 排列假设

MRIO 数据必须按连续区域块排列：

```text
区域1-部门1
区域1-部门2
...
区域2-部门1
区域2-部门2
...
```

并满足：

```text
区域数 × 每区域部门数 = n
```

各区域应使用相同的部门结构。当前界面对不相等配置只在浏览器控制台给出警告，不会阻止继续计算。

MRIO 模式下的核心矩阵计算仍对完整 $n\times n$ 系统执行。区域页目前只按连续切片汇总总产出、增加值和增加值率，不计算区域间隐含碳净转移或双边责任分配。

## 导入方式

| 格式 | 当前行为与要求 |
|---|---|
| `.xlsx`、`.xls` | 读取全部 Sheet，由用户为每个 Sheet 指定数据类型；可提取部门、最终需求、增加值、卫星账户和区域名称 |
| `.csv` | 每个文件读取一个矩阵；当前实现固定跳过首行和首列，因此文件应包含标题行和行标签列 |
| `.txt`、`.dat` | 每个文件读取一个矩阵；自动识别 Tab、分号、逗号或空格分隔，不自动跳过标题 |
| `.mat` | 支持部分未压缩 MAT v5 数值矩阵；不支持 MAT v7.3/HDF5 和压缩数据元素，建议从 MATLAB 导出为 `-v6` |
| 粘贴 | 支持 Tab、逗号或空白分隔；一次写入一个数据类型，可选择是否跳过首行和首列标题 |

对于 CSV、TXT、DAT 和 MAT，应用根据文件名或变量名推断数据类型。建议使用：

```text
Z
x
Y
VA
F
```

名称匹配采用简单的包含判断，复杂名称可能被误判。导入后必须检查维度预览。

### 推荐 Excel 工作簿结构

建议每类数据单独使用一个 Sheet，并在导入弹窗中手动确认类型：

```text
Z
x
Y
VA
F
sectors   （可选）
regions   （可选）
```

`x` Sheet 可以是一行、一列，或“部门名称 + x 数值”的两列表。勾选标题选项后，导入器会识别并保留行标签。

### 重要解析限制

- `x` 是必需输入；缺失时不会再从 `Z` 猜测或生成。
- 空单元格、非数值、`NaN`、`Infinity` 和不等长行会报告原始行列位置并阻止导入或计算。
- Tab 与 CSV 粘贴会保留空单元格的位置，不会把后续列左移。
- 数值解析要求整个单元格都是有限数值，`12abc` 不会被当作 `12` 接受。

应在导入前清理缺失值、文本标记和合并单元格，并始终提供真实总产出。

## 当前计算方法

记 $\hat{x}=\operatorname{diag}(x)$。零产出部门会在校验阶段阻止计算，不再静默把对应系数设为 `0`。

### 基础系数

直接消耗系数：

$$
A=Z\hat{x}^{-1},\qquad a_{ij}=\frac{z_{ij}}{x_j}
$$

分配系数：

$$
B=\hat{x}^{-1}Z,\qquad b_{ij}=\frac{z_{ij}}{x_i}
$$

Leontief 与 Ghosh 逆：

$$
L=(I-A)^{-1},\qquad G=(I-B)^{-1}
$$

### 最终需求与卫星账户

当前界面固定把最终需求各列合并为：

$$
y=Y\mathbf{1}
$$

直接卫星强度与足迹乘数：

$$
s=F\hat{x}^{-1},\qquad M=sL
$$

合并最终需求后的卫星总量：

$$
q=My
$$

当前界面默认合并最终需求列，计算引擎也支持保留每个最终需求类别。多卫星账户结果分别表示为 $p\times1$ 或 $p\times k$，并可在页面和导出文件中核对。

### 产业关联

后向与前向关联：

$$
BL_j=\sum_i L_{ij},\qquad FL_i=\sum_j G_{ij}
$$

两者分别除以全部部门的算术平均值得到标准化系数。当标准化后向和前向关联系数均大于 1 时，界面把该部门标记为关键产业。

## 部门合并

校验没有错误时，可启用部门合并。

### 等量合并

每连续 $N$ 个部门合并为一个部门。`Z` 按新行组与列组求和，`x` 和 `Y` 按部门行求和，`VA` 与 `F` 按部门列求和。

MRIO 数据只有在 $N$ 能整齐适配每区域部门结构时才应使用等量合并，否则可能跨越区域边界。

### 自定义合并

规则使用 1-based 部门编号：

```text
15,20=部门15+20
22,23,24=制造业合计
```

未出现在规则中的部门保持不变。MRIO 模式下，编号表示区域内部门位置，同一组规则会应用到每个区域。

合并直接替换当前内存中的数据。当前没有撤销按钮，需要重新加载原始文件或重置应用。

## 结果展示与导出

矩阵 `A`、`B`、`L`、`G`、`s` 和 `M` 支持表格与热力图切换。向量、乘数、最终需求足迹、产业关联和区域规模使用表格显示。

下载文件名形如：

```text
IO_Results_YYYYMMDD.xlsx
IO_Results_YYYYMMDD.json
```

当前页面、Excel 和 JSON 的指标覆盖并不完全一致：

| 内容 | 页面 | Excel | JSON |
|---|---:|---:|---:|
| 原始 `Z`、`x`、`Y`、`VA`、`F` | 仅维度摘要 | 是 | 否 |
| `A`、`L`、总增加值系数、乘数、`s`、`M` | 是 | 是 | 是 |
| `B` | 是 | 选择计算时导出 | 是 |
| `G` | 是 | 是 | 是 |
| 中间投入率、增加值率 | 否 | 是 | 是 |
| 最终需求足迹 | 是 | 是 | 是 |
| 产业关联与关键产业 | 是 | 是 | 是 |
| MRIO 区域规模 | 是 | 否 | 否 |

代码中有单矩阵 CSV 导出函数，但当前页面没有对应按钮。

## 隐私、网络与安全

- 文件由浏览器读取并在当前页面内计算，源码中没有把输入矩阵上传到应用服务器的请求。
- 当前代码不使用 `localStorage`、`sessionStorage` 或 IndexedDB 持久化数据。
- 下载结果由浏览器本地生成。
- 页面启动时会请求 Google Fonts 和 jsDelivr 上的 ECharts；当前版本不是完全离线封装。
- 页脚数据库链接只有在用户点击后才会访问对应网站。
- 导入的 Sheet、部门、区域、最终需求和卫星账户名称在进入页面或热力图 HTML 前会统一转义。
- CSV 标签会转义逗号、引号和换行，并中和以 `=`, `+`, `-`, `@` 开头的电子表格公式。
- Excel 读取仍依赖 npm 版 `xlsx`，其公开安全公告尚无 npm 修复版本；当前 alpha 仍应只处理可信文件。

## 已知限制

- MAT 支持范围有限，不支持 v7.3/HDF5 和压缩矩阵元素。
- MRIO 区域页只是通过维度校验后的描述性规模汇总，不是双边隐含流、净转移或消费责任分析。
- 未实现数据平衡、部门映射、单位转换、价格调整和数据库自动接入。
- 计算线程仍会显式求逆；尚未改为只求所需线性方程解，也没有稀疏矩阵后端。
- 浏览器 alpha 硬限制为 500 个部门；更大的 MRIO 必须使用后端或稀疏计算流程。
- 页面表格最多显示前 100 行和 100 列，热力图最多抽样为 200×200；导出保留完整结果。
- 页面、Excel、JSON 已统一主要计算结果，但原始输入和描述性区域汇总的覆盖仍不同。
- 步骤、上传区、结果标签页和导入弹窗已加入键盘与语义支持，但尚未完成完整屏幕阅读器审计。
- 依赖审计已从 7 项降至 1 项高危告警；剩余项来自 npm 版 `xlsx`，没有自动修复版本。
- 没有项目保存、撤销、版本管理或协作功能。
